# Data Structure & 3rd-Party Usage

A core goal of Baumradar is to eliminate the tedious processing of raw Open Data for other developers. You can directly use the cleaned, processed SQLite databases generated by our backend in your own projects (e.g., iOS app, web app, data analysis, school project).

---

## Step 1: Finding the Data

All data is accessible via a central JSON file:

```
https://raw.githubusercontent.com/matthili/BaumRadar/master/docs/data/catalog.json
```

This file has the following structure:

```json
{
  "version": 1,
  "cities": [
    {
      "id": "vienna",
      "name": "Wien",
      "country": "Österreich",
      "boundingBox": [48.12, 16.18, 48.32, 16.58],
      "dbUrl": "https://raw.githubusercontent.com/.../vienna.db.gz",
      "sigUrl": "https://raw.githubusercontent.com/.../vienna.db.gz.sig"
    },
    {
      "id": "berlin",
      "name": "Berlin",
      "country": "Deutschland",
      "boundingBox": [52.33, 13.08, 52.68, 13.76],
      "dbUrl": "https://raw.githubusercontent.com/.../berlin.db.gz",
      "dbUrlChunks": [
        "https://...berlin.db.gz.001",
        "https://...berlin.db.gz.002"
      ],
      "sigUrl": "https://raw.githubusercontent.com/.../berlin.db.gz.sig"
    }
  ]
}
```

**Fields explained:**
- `id`: Unique identifier for the city.
- `boundingBox`: `[minLat, minLon, maxLat, maxLon]` – the geographic extent of the city.
- `dbUrl`: Download URL for the compressed database. **Note:** For cities with `dbUrlChunks`, you must use the chunks (the `dbUrl` may point to a non-existent file).
- `dbUrlChunks`: If the database exceeds 50MB (e.g., Berlin), it is split into multiple parts. These must be concatenated at the binary level in order.
- `sigUrl`: URL of the Ed25519 signature.

---

## Step 2: Download & Extract

### For files without chunks:
1. Download `dbUrl` → e.g., `vienna.db.gz`
2. Decompress the GZIP file → `vienna.db`

### For files with chunks (e.g., Berlin):
1. Download all chunk URLs in order (`.001`, `.002`, ...)
2. Concatenate the files at the binary level → `berlin.db.gz`
3. Decompress the GZIP file → `berlin.db`

---

## Step 3: Verify Signature (optional, but recommended)

To ensure the data is authentic and has not been tampered with, every database is signed by the backend using Ed25519.

1. Download the **Ed25519 Public Key** from the repository: `docs/data/public_key.b64` (Base64-encoded, X.509 format).
2. Download the signature file (e.g., `vienna.db.gz.sig`).
3. Verify the `.db.gz` file (or the reassembled chunks, **before** decompression) against the signature.

**Note on chunks:** The signature applies to the reassembled `.db.gz` file, not to individual chunks. You must first concatenate all chunks, then verify the complete file against the signature.

## The SQLite Database

The extracted `.db` file is a standard SQLite database that can be opened with any SQLite library. It contains two tables:

### Table `trees`

Contains every individual tree as its own row.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT (PK) | Unique ID of the tree |
| `city_id` | TEXT | ID of the city (e.g., `"vienna"`) |
| `lat` | REAL | WGS84 Latitude |
| `lon` | REAL | WGS84 Longitude |
| `genus_de` | TEXT | German genus name (e.g., `"Birke"`) |
| `genus_en` | TEXT | English genus name (e.g., `"Birch"`) |
| `species_de` | TEXT | German species name (e.g., `"Hänge-Birke"`), may be NULL |
| `species_en` | TEXT | English species name (e.g., `"Silver Birch"`), may be NULL |

### Table `geofences`

Pre-calculated zones for tree groups of the same genus. A geofence merges multiple closely-spaced trees into a single circle. This saves immense computation time for route-collision checks.

| Column | Type | Description |
|---|---|---|
| `id` | TEXT (PK) | Unique zone ID (UUID) |
| `lat` | REAL | WGS84 Latitude of the zone center |
| `lon` | REAL | WGS84 Longitude of the zone center |
| `radius` | INTEGER | Radius in meters (50m for individual trees, 100m for groups) |
| `count` | INTEGER | Number of trees in this cluster |
| `genus_de` | TEXT | **The relation to `trees`!** German genus name – corresponds to `trees.genus_de` |

**How are the tables related?**
The column `geofences.genus_de` contains exactly the same value as `trees.genus_de`. If a user selects an allergy to `"Birke"` (Birch), you can query both the individual trees (`SELECT * FROM trees WHERE genus_de = 'Birke'`) and the pre-calculated zones (`SELECT * FROM geofences WHERE genus_de = 'Birke'`).

## Code Examples

### Python: Find all Birch trees in Vienna

```python
import sqlite3
import gzip
import urllib.request

# 1. Download and extract the database
urllib.request.urlretrieve(
    "https://raw.githubusercontent.com/matthili/BaumRadar/master/docs/data/vienna.db.gz",
    "vienna.db.gz"
)

with gzip.open("vienna.db.gz", "rb") as f_in:
    with open("vienna.db", "wb") as f_out:
        f_out.write(f_in.read())

# 2. Open and query the database
conn = sqlite3.connect("vienna.db")
cursor = conn.cursor()

# All Birch trees in Vienna
cursor.execute("""
    SELECT id, lat, lon, genus_de, species_de 
    FROM trees 
    WHERE genus_de = 'Birke'
""")

birches = cursor.fetchall()
print(f"Number of Birch trees in Vienna: {len(birches)}")

for tree in birches[:5]:
    print(f"  ID: {tree[0]}, Position: ({tree[1]:.5f}, {tree[2]:.5f}), Species: {tree[4] or 'unknown'}")

# Geofence zones for Birch
cursor.execute("""
    SELECT lat, lon, radius, count 
    FROM geofences 
    WHERE genus_de = 'Birke'
""")

zones = cursor.fetchall()
print(f"\nNumber of Birch hotspots: {len(zones)}")

for zone in zones[:3]:
    print(f"  Center: ({zone[0]:.5f}, {zone[1]:.5f}), Radius: {zone[2]}m, Trees: {zone[3]}")

conn.close()
```

### Python: List all tree species

```python
import sqlite3

conn = sqlite3.connect("vienna.db")
cursor = conn.cursor()

cursor.execute("""
    SELECT genus_de, genus_en, COUNT(*) as count 
    FROM trees 
    WHERE genus_de IS NOT NULL AND genus_de != '' 
    GROUP BY genus_de 
    ORDER BY count DESC
""")

print("Tree species in Vienna:")
print(f"{'German':<20} {'English':<20} {'Count':>8}")
print("-" * 50)

for row in cursor.fetchall():
    print(f"{row[0]:<20} {row[1] or '?':<20} {row[2]:>8}")

conn.close()
```

### Kotlin (Android): Find nearby trees

```kotlin
// If you want to use the database directly in an Android app:
val db = SQLiteDatabase.openDatabase("path/to/vienna.db", null, SQLiteDatabase.OPEN_READONLY)

val cursor = db.rawQuery("""
    SELECT id, lat, lon, genus_de, species_de 
    FROM trees 
    WHERE lat BETWEEN ? AND ? AND lon BETWEEN ? AND ?
""", arrayOf(
    (myLatitude - 0.005).toString(),    // approx. 500m south
    (myLatitude + 0.005).toString(),    // approx. 500m north
    (myLongitude - 0.007).toString(),   // approx. 500m west
    (myLongitude + 0.007).toString()    // approx. 500m east
))

while (cursor.moveToNext()) {
    val name = cursor.getString(3)   // genus_de
    val lat  = cursor.getDouble(1)
    val lon  = cursor.getDouble(2)
    println("$name at ($lat, $lon)")
}
cursor.close()
db.close()
```

### JavaScript (Node.js): Analyze the database

```javascript
// npm install better-sqlite3
const Database = require('better-sqlite3');
const db = new Database('vienna.db', { readonly: true });

// Top 10 most common tree species
const rows = db.prepare(`
    SELECT genus_de, genus_en, COUNT(*) as count 
    FROM trees 
    WHERE genus_de IS NOT NULL 
    GROUP BY genus_de 
    ORDER BY count DESC 
    LIMIT 10
`).all();

console.log('Top 10 tree species in Vienna:');
rows.forEach(row => {
    console.log(`  ${row.genus_de} (${row.genus_en}): ${row.count} trees`);
});

// Geofence zones for allergy sufferers
const allergen = 'Hasel';
const zones = db.prepare(`
    SELECT lat, lon, radius, count 
    FROM geofences 
    WHERE genus_de = ?
`).all(allergen);

console.log(`\n${allergen} hotspots: ${zones.length} zones`);
db.close();
```

## Usage Summary

1. Fetch the `catalog.json` (see Step 1).
2. Download and extract the desired city database (Step 2).
3. Optional: Verify the signature (Step 3).
4. Open the `.db` file with any SQLite library of your choice.
5. Query as shown in the code examples above.
